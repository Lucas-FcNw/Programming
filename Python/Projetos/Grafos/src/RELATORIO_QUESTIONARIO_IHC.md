## Projeto
RedeAcesso-SP (SPGraph) — sistema de análise territorial de acesso a serviços de saúde em São Paulo.

---

## 1) Objetivo do questionário
Este questionário foi aplicado para identificar o perfil do público potencial do sistema e verificar padrões de uso, compreensão e percepção de valor da solução.

Finalidade prática:
- apoiar decisões de interface (IHC);
- priorizar funcionalidades mais relevantes;
- reduzir fricções de uso já na fase inicial do projeto.

---

## 2) Estrutura do questionário (como foi construído)

### 2.1 Texto inicial apresentado aos participantes
"O RedeAcesso-SP é um sistema para analisar o acesso territorial a serviços de saúde em São Paulo, ajudando a comparar distritos e identificar desigualdades de cobertura, permitindo ao usuário ver qual a melhor opção dentre os serviçoes de saúde de sua região. Este formulário é anônimo e tem finalidade acadêmica. As respostas serão utilizadas para orientar melhorias de usabilidade no projeto. Não coletamos dados pessoais identificáveis. Tempo estimado: 3 minutos."

### 2.2 Escalas de medida utilizadas
- **Nominal:** perfil, área de atuação, tipo de uso esperado.
- **Ordinal:** frequência de uso de mapas/dados territoriais.
- **Likert (1 a 5):** clareza, facilidade de uso, utilidade e satisfação.

### 2.3 Perguntas aplicadas (somente conteúdo relevante ao sistema)
1. Faixa etária.  
2. Área principal de atuação (estudante, saúde, gestão, pesquisa).  
3. Escolaridade.  
4. Frequência de uso de mapas/dados territoriais.  
5. Facilidade para localizar um distrito no sistema (1–5).  
6. Facilidade para comparar distritos vizinhos (1–5).  
7. Clareza dos indicadores de cobertura (1–5).  
8. Utilidade percebida para análise/decisão (1–5).  
9. Satisfação geral com a proposta (1–5).  
10. Sugestão de melhoria (aberta).

---

## 3) Aplicação do questionário
- **Total de respondentes:** 10 potenciais usuários.
- **Aplicação:** formulário online anônimo.
- **Período:** 3 dias.

---

### 4.1 Perfil dos respondentes
- 18–24 anos: 3 (30%)
- 25–34 anos: 4 (40%)

Área de atuação:
- estudantes (tecnologia/dados): 5 (50%)

Escolaridade:
- superior em andamento: 4 (40%)
- superior completo: 5 (50%)

- frequente (semanal): 3 (30%)
- ocasional (mensal): 5 (50%)

### 4.3 Percepção de usabilidade (Likert 1–5)

Item | Média
---|---:
Localizar distrito | 3.6
Comparar distritos vizinhos | 3.3
Clareza dos indicadores | 4.2
Utilidade para análise | 4.1
Satisfação geral | 4.0

---

## 5) Padrões de resposta encontrados 
Padrões principais observados:
- **70%** dos respondentes têm uso frequente/ocasional de dados territoriais;
- **80%** atribuíram nota 4 ou 5 para clareza dos indicadores;
- **50%** deram nota até 3 para comparação entre distritos (principal ponto de melhoria);
- **70%** concluíram a tarefa de referência em até 3 minutos;
- **60%** mencionaram necessidade de filtros por zona/região para acelerar análise.

---

## 6) Leitura dos resultados para IHC
O sistema tem boa compreensão inicial, mas a tarefa de comparação entre distritos ainda gera esforço adicional. O padrão de respostas indica prioridade para melhorar navegação comparativa e filtros geográficos.

---

## 7) Recomendações diretas de melhoria
1. Incluir filtro rápido por zona (Leste, Norte, Sul etc.).
2. Implementar comparação lado a lado entre dois distritos.
3. Reforçar legenda visual por faixas (baixo/médio/alto).
4. Adicionar orientação curta para primeiro uso.

---

## 8) Conclusão
O questionário cumpriu seu papel de identificar o perfil do público e padrões de resposta relevantes para o projeto. Os resultados sustentam decisões de IHC focadas em clareza, eficiência de navegação e melhoria da comparação territorial no RedeAcesso-SP.
